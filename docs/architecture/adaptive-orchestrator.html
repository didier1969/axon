Axon Control Plane Proposal

Adaptive Orchestrator

L’orchestrateur adaptatif agit comme un coordinateur en boucle fermée. Il observe l’état réel du pipeline d’ingestion, la santé du serveur MCP et la pression des ressources, puis ouvre ou ferme progressivement les vannes du système pour maximiser la complétion du backlog sans casser l’usage interactif.

Objectif principal Max backlog drain
Contrainte mémoire ≥ 33% RAM libre
Contrainte interactive MCP ≤ 300 ms
Contrainte CPU Axon ≤ 50%
Flux entrants

Coordinateur central

Adaptive Orchestrator

Boucle fermée multi-objectifs. À chaque cycle, il observe le système, vérifie les contraintes dures, estime la marge de manœuvre, puis ajuste les paramètres de débit.

Priorité 1: maximiser backlog_drain_rate
Sous contraintes:
mem_available_ratio ≥ 0.33
mcp_latency_p95 ≤ 300ms
cpu_usage_ratio ≤ 0.50
Logique initiale
1. Observer

Collecter backlog, pression service, coûts embed, disponibilité mémoire, budget CPU.

2. Vérifier

Si une contrainte dure casse, réduire rapidement le débit ou passer en mode gardé.

3. Ouvrir lentement

Si tout est stable, augmenter progressivement les vannes sur une seule dimension à la fois.

4. Refermer vite

En cas de dérive latence/RAM/CPU, réduire immédiatement les paramètres récemment ouverts.

États de décision
Quiet Cruise Aggressive Drain Recovery Interactive Guarded Gpu Scaling Blocked
