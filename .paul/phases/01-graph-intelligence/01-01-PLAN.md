<objective>
## Goal
Close parser gaps: extract type_refs from TS class properties + interface members (so USES_TYPE edges are created for those contexts), fix Python `from x import *` IMPORTS edge, and add Go `_test.go` exempt pattern to dead code.

## Purpose
The USES_TYPE graph edges are incomplete for TypeScript codebases — class property annotations (`items: Array<User>`) and interface member types (`bar: Map<string, Entity>`) currently produce no type_refs, so those type relationships are invisible to `axon_dead_code` and `axon_context`. Wildcard Python imports create an ImportInfo entry but the IMPORTS edge may silently drop when the module resolves to None. Go test files (`foo_test.go`) are incorrectly flagged as dead code candidates.

## Output
- Modified: typescript.py, python_lang.py, dead_code.py
- Verified by: test suite (no regressions), new targeted tests
</objective>
