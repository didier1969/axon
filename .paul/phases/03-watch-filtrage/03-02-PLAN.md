<objective>
## Goal
Refactor `watch_repo()` to use a producer/consumer pattern with `asyncio.Queue`: the `awatch` loop puts path batches into the queue (non-blocking), while a separate consumer coroutine drains the queue sequentially — one batch at a time.

## Purpose
The current loop interleaves event collection and processing: it awaits `_reindex_files` before fetching the next batch from `watchfiles.awatch`. In `serve --watch` mode with a lock, if MCP holds the lock during a query, the producer stalls and new filesystem events can't even be collected. With a queue, the producer always runs freely — events are buffered — and the consumer processes them sequentially without blocking the producer.

## Output
- `watch_repo()` refactored with inner `_producer()` and `_consumer()` coroutines
- `asyncio.Queue[list[Path] | None]` used as the handoff channel; `None` is the sentinel
- Timer logic (dirty, last_global, last_embed) moves to the consumer
- No change to public signature or behaviour from the caller's perspective
- 2 new tests confirming multi-batch processing and sequential consumption
</objective>

    **Step 2 — Inner `_producer()` coroutine:**
    Move the `awatch` loop into `_producer`. It puts `list[Path]` batches into the queue.
    After the loop exits (stop_event fired or awatch exhausted), it puts `None` (sentinel).
    ```python
    async def _producer() -> None:
        async for changes in watchfiles.awatch(
            repo_path,
            rust_timeout=debounce_ms,
            watch_filter=_make_watch_filter(repo_path, gitignore),
            stop_event=stop_event,
            ignore_permission_denied=True,
        ):
            changed_paths: list[Path] = []
            seen: set[str] = set()
            for _change_type, path_str in changes:
                if path_str not in seen:
                    seen.add(path_str)
                    changed_paths.append(Path(path_str))
            if changed_paths:
                await queue.put(changed_paths)
        await queue.put(None)  # signal consumer to stop
    ```

    **Step 3 — Inner `_consumer()` coroutine:**
    Move all processing logic (reindex, timers, dirty flag) into `_consumer`.
    ```python
    async def _consumer() -> None:
        dirty = False
        last_global = time.monotonic()
        last_embed = time.monotonic()
        files_changed = 0

        while True:
            batch = await queue.get()
            if batch is None:
                queue.task_done()
                break

            count = await _run_sync(_reindex_files, batch, repo_path, storage, gitignore)
            if count > 0:
                files_changed += count
                dirty = True
                logger.info("Reindexed %d file(s)", count)

            now = time.monotonic()
            if dirty and (now - last_global) >= GLOBAL_PHASE_INTERVAL:
                with_embeddings = (now - last_embed) >= EMBEDDING_INTERVAL
                logger.info(
                    "Running global analysis phases (embeddings=%s)...", with_embeddings
                )
                await _run_sync(_run_global_phases, storage, repo_path, with_embeddings)
                dirty = False
                last_global = now
                if with_embeddings:
                    last_embed = now

            queue.task_done()

        logger.info("Watch stopped. Total files reindexed: %d", files_changed)
    ```

    **Step 4 — Main body of watch_repo (after _run_sync, before return):**
    Replace the old `async for` loop body with:
    ```python
    logger.info("Watching %s for changes...", repo_path)
    await asyncio.gather(
        asyncio.create_task(_producer()),
        asyncio.create_task(_consumer()),
    )
    ```

    **Important:**
    - `_run_sync`, `_producer`, `_consumer` are all inner functions/coroutines inside `watch_repo` — they close over `queue`, `repo_path`, `storage`, `gitignore`, `debounce_ms`, `stop_event`, `lock`
    - The `asyncio.Queue` type annotation uses `list[Path] | None` — ensure Python 3.10+ union syntax or use `Optional` if needed (check existing codebase style)
    - Do NOT use `queue.join()` — the sentinel pattern is sufficient
    - Do NOT change the module-level `logger.info("Watching %s...")` position — it now lives inside `watch_repo` before `asyncio.gather`

    ```python
    class TestWatchRepoQueue:
        """watch_repo() processes all batches via internal asyncio.Queue."""

        def test_processes_multiple_batches(
            self, tmp_repo: Path, storage: KuzuBackend
        ) -> None:
            import asyncio
            from unittest.mock import patch, MagicMock

            call_batches: list[list[Path]] = []
            _real_reindex = _reindex_files

            def tracking_reindex(paths: list[Path], *args, **kwargs) -> int:
                call_batches.append(list(paths))
                return _real_reindex(paths, *args, **kwargs)

            async def fake_awatch(*args, **kwargs):
                # Yield two distinct batches, then exit (simulates stop_event/end of stream)
                yield {(MagicMock(), str(tmp_repo / "src" / "app.py"))}
                yield {(MagicMock(), str(tmp_repo / "src" / "utils.py"))}

            with patch("axon.core.ingestion.watcher._reindex_files", tracking_reindex):
                with patch("watchfiles.awatch", fake_awatch):
                    asyncio.run(watch_repo(tmp_repo, storage))

            # Both batches must be processed, each in a separate _reindex_files call
            assert len(call_batches) == 2
            paths_seen = {str(p) for batch in call_batches for p in batch}
            assert str(tmp_repo / "src" / "app.py") in paths_seen
            assert str(tmp_repo / "src" / "utils.py") in paths_seen

        def test_empty_changes_not_queued(
            self, tmp_repo: Path, storage: KuzuBackend
        ) -> None:
            import asyncio
            from unittest.mock import patch, MagicMock

            call_count: list[int] = []

            def counting_reindex(paths: list[Path], *args, **kwargs) -> int:
                call_count.append(len(paths))
                return 0

            async def fake_awatch(*args, **kwargs):
                # Only ignored file — deduplicated to empty set, should NOT be queued
                yield {(MagicMock(), str(tmp_repo / ".paul" / "STATE.md"))}
                yield {(MagicMock(), str(tmp_repo / "src" / "app.py"))}

            with patch("axon.core.ingestion.watcher._reindex_files", counting_reindex):
                with patch("watchfiles.awatch", fake_awatch):
                    asyncio.run(watch_repo(tmp_repo, storage))

            # .paul/STATE.md is pre-filtered by watch_filter before entering queue
            # app.py is not ignored — exactly 1 call (for app.py batch only)
            # Note: if watch_filter filters .paul at watchfiles level, only app.py batch enters queue
            assert len(call_count) == 1
    ```
