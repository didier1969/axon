<objective>
## Goal
Add `start_byte` and `end_byte` fields to `SymbolInfo`, `GraphNode`, and the KuzuDB schema. Propagate the values from all tree-sitter parsers (where `node.start_byte` / `node.end_byte` are available). This enables O(1) symbol content retrieval via `file_bytes[start_byte:end_byte]` without re-parsing.

## Purpose
All tree-sitter parsers already compute `content = file_bytes[node.start_byte:node.end_byte]` but discard the raw offsets. Storing them in the graph enables callers to retrieve exact source bytes by position without loading and scanning the file. This is the foundation for future content retrieval tools and caching optimisations.

## Output
- `SymbolInfo.start_byte` / `.end_byte` (int, default 0)
- `GraphNode.start_byte` / `.end_byte` (int, default 0)
- KuzuDB node tables gain `start_byte INT64, end_byte INT64` columns
- 8 tree-sitter parsers set the fields on every `SymbolInfo` they emit
- 1 new test confirms a parsed Python function carries non-zero byte offsets
</objective>

    **Pattern for all tree-sitter parsers:**
    Every parser already computes `node_content = content[node.start_byte:node.end_byte]` or uses `node.start_point` etc., so `node.start_byte` and `node.end_byte` are always in scope wherever `SymbolInfo(...)` is called.

    ```python
    # BEFORE:
    result.symbols.append(
        SymbolInfo(
            name=name,
            kind=kind,
            start_line=start_line,
            end_line=end_line,
            content=node_content,
            ...
        )
    )

    # AFTER:
    result.symbols.append(
        SymbolInfo(
            name=name,
            kind=kind,
            start_line=start_line,
            end_line=end_line,
            start_byte=node.start_byte,
            end_byte=node.end_byte,
            content=node_content,
            ...
        )
    )
    ```
