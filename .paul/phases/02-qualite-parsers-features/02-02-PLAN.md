<objective>
## Goal
Fix three parser quality gaps: dead code test-file patterns, Python wildcard imports, and TypeScript generic type arguments. Also close the test coverage gap from 02-01 (SQL/YAML byte offsets + axon_read_symbol had no tests).

## Purpose
- `_is_test_file()` misses common patterns (`spec/`, `__tests__/`, `_spec.rb`, `.spec.ts`, `.test.js`) — symbols in spec files are incorrectly flagged as dead code
- `from x import *` in Python generates `ImportInfo(names=[])` — the wildcard edge is silently dropped
- TypeScript generic annotations (`:Array<User>`, `:Promise<Data>`) return `""` from `_type_annotation_name()` — zero USES_TYPE edges for generic types; type parameter constraints (`<T extends Schema>`) are also skipped
- 02-01 shipped SQL/YAML byte offsets and `axon_read_symbol` with no new tests — coverage debt must be closed

## Output
- `dead_code.py`: `_is_test_file()` with 6 additional patterns
- `python_lang.py`: `_extract_import_from()` handles `wildcard_import` node
- `typescript.py`: `_type_annotation_name()` handles `generic_type`; new `_extract_generic_arg_refs()` + `_extract_type_param_constraints()` helpers; callers updated
- New test assertions in 5 test files (≥852 + ~20 new tests total)
</objective>

    **2. Add `_extract_generic_arg_refs()` static method:**

    New method that extracts type arguments from a `generic_type` node inside a `type_annotation`:
    ```python
    @staticmethod
    def _extract_generic_arg_refs(
        annotation_node: Node, kind: str, line: int, param_name: str = ""
    ) -> list[TypeRef]:
        """Extract TypeRefs for type arguments inside generic annotations like Array<User>."""
        refs: list[TypeRef] = []
        for child in annotation_node.children:
            if child.type == "generic_type":
                type_args = child.child_by_field_name("type_arguments")
                if type_args is not None:
                    for arg in type_args.children:
                        if arg.type in ("type_identifier", "identifier"):
                            name = arg.text.decode()
                            if name.lower() not in _BUILTIN_TYPES:
                                refs.append(TypeRef(
                                    name=name, kind=kind, line=line, param_name=param_name,
                                ))
        return refs
    ```

    **3. Add `_extract_type_param_constraints()` static method:**

    Extracts constraint types from `type_parameters` (e.g., `<T extends Schema>` → `Schema`):
    ```python
    @staticmethod
    def _extract_type_param_constraints(
        type_params_node: Node, result: ParseResult
    ) -> None:
        """Extract TypeRefs for constraints in type parameters like <T extends Schema>."""
        for child in type_params_node.children:
            if child.type == "type_parameter":
                constraint = child.child_by_field_name("constraint")
                if constraint is None:
                    continue
                for sub in constraint.children:
                    if sub.type in ("type_identifier", "identifier"):
                        name = sub.text.decode()
                        if name.lower() not in _BUILTIN_TYPES:
                            result.type_refs.append(TypeRef(
                                name=name,
                                kind="param",
                                line=sub.start_point[0] + 1,
                            ))
    ```

    **4. Call the new helpers from existing extraction methods:**

    In `_extract_function_types()`:
    - For each param's `type_annotation` sub, after creating the TypeRef, also call `_extract_generic_arg_refs(sub, "param", line, param_name)` and append results to `result.type_refs`
    - For return type `type_annotation`, similarly call `_extract_generic_arg_refs(child, "return", line)`
    - After handling params, get `type_parameters = func_node.child_by_field_name("type_parameters")` and call `_extract_type_param_constraints(type_parameters, result)` if not None

    In `_extract_variable_type_annotation()`:
    - After the existing TypeRef append, also call `_extract_generic_arg_refs(child, "variable", line)` and extend `result.type_refs`

    In `_extract_class()`:
    - After `result.symbols.append(...)`, get `type_parameters = node.child_by_field_name("type_parameters")` and call `_extract_type_param_constraints(type_parameters, result)` if not None

    In `_extract_interface()`:
    - Same: call `_extract_type_param_constraints` on `type_parameters` field if present

    In `_extract_type_alias()`:
    - Same: call `_extract_type_param_constraints` on `type_parameters` field if present

    Avoid: changing the existing TypeRef for the base type (Array, Map, etc.) — keep extracting it as before. Only ADD the new refs; don't modify existing logic.
    Avoid: extracting the type variable itself (T, U) as a TypeRef — it's a local binding, not a real type.
