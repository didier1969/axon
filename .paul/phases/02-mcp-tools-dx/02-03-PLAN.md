<objective>
## Goal
Add `axon_summarize` MCP tool that produces a structured, LLM-ready summary of a file or symbol (class/module) from graph data — without making any LLM API calls itself.

## Purpose
AI agents frequently need to understand the shape of a file or class before diving into implementation details. Reading entire files is expensive and noisy. `axon_summarize` provides a curated structural overview — symbol inventory, method counts, tested/exported flags, centrality — that gives agents exactly what they need to plan work without wasting context tokens on boilerplate.

## Output
- Modified: mcp/tools.py (new handle_summarize), mcp/server.py (registration)
- New tests: tests/mcp/test_tools.py (file summary, class summary, not-found)
</objective>

```gherkin
Given a path "src/parsers/python.py" that has a FILE node with 3 CLASS and 8 FUNCTION children
When axon_summarize is called with path="src/parsers/python.py"
```

    Add `handle_summarize(storage, path, repo=None)` after `handle_lint`.

    Pattern: follow handle_context / handle_find_similar for repo loading and try/finally cleanup.

    The `path` argument is ambiguous — it could be a file path ("src/foo.py") or a symbol name ("MyClass"). Resolution order:

    **Step 1: Try file path lookup first.**
    ```python
    file_rows = storage.execute_raw(
        "MATCH (n:File) WHERE n.file_path = $fp OR n.file_path ENDS WITH $fp "
        "RETURN n.id, n.name, n.file_path, n.language "
        "LIMIT 1",
        parameters={"fp": path},
    ) or []
    ```
    If file_rows has a result → produce FILE SUMMARY (see below).

    **Step 2: If no file found, try symbol lookup.**
    ```python
    results = _resolve_symbol(storage, path, limit=1)
    ```
    If results empty → return "'{path}' not found as a file path or symbol name."
    If results found → produce SYMBOL SUMMARY (see below).

    ---

    **FILE SUMMARY** — when a File node is found:

    Get child symbols for the file:
    ```python
    children_rows = storage.execute_raw(
        "MATCH (n) WHERE n.file_path = $fp AND n.start_line > 0 "
        "RETURN n.id, n.name, n.file_path, n.start_line, n.signature, n.tested, n.is_exported, n.centrality "
        "ORDER BY n.start_line",
        parameters={"fp": file_path},
    ) or []
    ```

    Separate children by label prefix (from node_id: "class:", "function:", "method:", "interface:", etc.).

    Group into: classes, functions, interfaces, other.
    For each class, count how many rows have label "method" and class_name == class.name (skip this detail if complex — just show count from a sub-query or from children filtered by class_name field).

    Format:
    ```
    # src/parsers/python.py — 42 symbols
    Language: python

    ## Classes (3)
    1. PyParser  src/parsers/python.py:L10  [exported] [tested]
    2. NodeVisitor  src/parsers/python.py:L88
    3. ...

    ## Functions (8)
    1. parse_file  def parse_file(path: Path) -> list[GraphNode]  centrality: 0.12
    2. _extract_imports  [untested]
    ...

    ## Interfaces (0)

    Next: Use axon_context('ClassName') for the full dependency graph of any symbol.
    ```

    Only show sections that have at least 1 entry. Truncate each section to 10 items max.

    ---

    **SYMBOL SUMMARY** — when a symbol (class/function/method) is found via _resolve_symbol:

    Get full node:
    ```python
    node = storage.get_node(results[0].node_id)
    ```

    Get callers and callees counts (use get_callers_with_confidence and get_callees_with_confidence):
    ```python
    try:
        callers = storage.get_callers_with_confidence(node.id)
    except (AttributeError, TypeError):
        callers = [(c, 1.0) for c in storage.get_callers(node.id)]
    ```

    For CLASS nodes, also fetch method children:
    ```python
    method_rows = storage.execute_raw(
        "MATCH (n:Method) WHERE n.class_name = $cn AND n.file_path = $fp "
        "RETURN n.name, n.signature LIMIT 20",
        parameters={"cn": node.name, "fp": node.file_path},
    ) or []
    ```

    Format:
    ```
    # MyParser (Class) — src/parsers/python.py:L10

    Signature: class MyParser(BaseParser):
    tested: yes | exported: yes | centrality: 0.12

    Methods (5):
      parse(), _extract_imports(), _extract_calls(), visit_node(), __init__()

    Callers: 12 symbols call this
    Callees: 5 symbols called by this

    Next: Use axon_context('MyParser') for the full caller/callee list.
    ```

    For FUNCTION/METHOD nodes, skip the Methods section (not applicable).

    Call `log_event("summarize", path=path[:200], repo=...)` at end.

    Avoid: making any LLM API calls. Avoid: loading file contents from disk (use only graph data). Avoid: changing the storage schema — all data is already in the graph from existing ingestion phases.
    Avoid: N+1 queries — use one execute_raw for children, not one query per child.
