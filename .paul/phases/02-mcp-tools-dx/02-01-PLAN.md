<objective>
## Goal
Add `axon_find_usages` MCP tool that exhaustively lists all call-sites of a symbol across the repo, and improve MCP tool descriptions with concrete parameter formats and usage guidance for every existing tool.

## Purpose
`axon_context` shows callers but is limited to `_MAX_RELATIONS_DISPLAYED=20`. `axon_find_usages` provides an exhaustive, pageable list of every location where a symbol is called or imported — crucial for refactoring, dependency audits, and understanding impact before deletion. Improved descriptions reduce AI agent hallucinations and misuse of tool parameters (e.g. agents passing a node_id where a name is expected).

## Output
- Modified: mcp/tools.py (new handle_find_usages), mcp/server.py (registration + improved descriptions)
- New tests: tests/mcp/test_tools.py (find_usages: CALLS edges, IMPORTS edges, empty, not-found)
</objective>

  <name>Task 1: Implement handle_find_usages in tools.py</name>
  <files>src/axon/mcp/tools.py</files>
  <action>
    Add `handle_find_usages(storage, symbol, limit=50, repo=None)` after `handle_find_similar`.

    Pattern: follow handle_context / handle_find_similar exactly for repo loading and try/finally cleanup.

    Implementation steps:
    1. If repo provided: load via _load_repo_storage, return error if missing.
    2. Resolve symbol: `results = _resolve_symbol(storage, symbol, limit=1)` — if empty, return "Symbol '{symbol}' not found."
    3. Get node_id from results[0].node_id. Get node for display name and file.
    4. Query CALLS edges (incoming — symbols that call this one):
       ```python
       calls_rows = storage.execute_raw(
           "MATCH (caller)-[r:CodeRelation]->(callee) "
           "WHERE callee.id = $nid AND r.rel_type = 'calls' "
           "RETURN caller.name, caller.file_path, caller.start_line "
           f"LIMIT {limit}",
           parameters={"nid": node_id},
       ) or []
       ```
    5. Query IMPORTS edges (incoming — files that import this symbol's file):
       Get node file_path from results[0].file_path, then:
       ```python
       imports_rows = storage.execute_raw(
           "MATCH (importer)-[r:CodeRelation]->(imported) "
           "WHERE imported.file_path = $fp AND r.rel_type = 'imports' "
           "RETURN importer.name, importer.file_path, importer.start_line "
           f"LIMIT {limit}",
           parameters={"fp": results[0].file_path},
       ) or []
       ```
       Deduplicate importers: keep only unique file_path values (one entry per importing file).
    6. If both empty: return "No usages found for '{symbol}'."
    7. Format:
       ```
       N call sites for 'parse_file' (src/parsers/python.py):

       CALLS (N):
         1. caller_name  src/pipeline.py:L42
         2. ...

       IMPORTS (N):
         1. src/cli.py
         2. ...

       Next: Use axon_impact('parse_file') for transitive blast radius.
       ```
    8. Call `log_event("find_usages", ...)` at end.

    Avoid: string interpolation of user input into Cypher — use `parameters={}` for all values.
    Avoid: duplicating the repo-loading try/finally boilerplate — follow the exact same pattern as handle_find_similar (lines ~768-841 in tools.py).
