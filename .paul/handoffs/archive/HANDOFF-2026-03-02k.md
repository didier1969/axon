## Architecture Context

```
axon_batch design:
  call_tool("axon_batch", {calls: [...], max_tokens: N})
    → _batch_daemon_call(calls, max_tokens)   [ONE socket, N sequential send/recv]
      → success: formatted result ("### tool (i/N)\n...")
      → fail (no sock / OSError): None
    → fallback: iterate calls, _dispatch_tool() for each
    → max_tokens applied per sub-result (NOT per total)

Key: daemon unchanged — receives individual tool calls, never "axon_batch"
```

**_batch_daemon_call implementation:**
- Open 1 socket to `~/.axon/daemon.sock`
- For each call: strip `repo` from daemon_args (slug from `args.get("repo")`), `encode_request`, `recv` until `\n`
- Daemon's `_handle_connection` already handles multiple requests on one connection (while True loop)
- Result format: `"### axon_query (1/3)\nresult\n\n### axon_context (2/3)\nresult"`
