## Architecture v0.6 cible (contexte essentiel)

**Problème résolu :** 1 axon pour tous les projets, sobre en ressources

```
Aujourd'hui (❌)                    V0.6 (✅)
Claude Code A                       Claude Code A
└─ axon mcp (1-4 GB)               └─ axon mcp proxy (10 MB)
   └─ KuzuDB nexus                              ↓
   └─ embeddings model              ┌─────────────────────┐
                                    │  axon daemon (1x)   │
Claude Code B                       │  - 1 embeddings     │
└─ axon mcp (1-4 GB)               │  - LRU DB cache (5) │
   └─ KuzuDB trader                 │  - Unix socket      │
                                    └─────────────────────┘
12 watchers × 1-4GB = chaos
```

**Constraint clé :** KuzuDB single-writer lock → le daemon le résout naturellement
