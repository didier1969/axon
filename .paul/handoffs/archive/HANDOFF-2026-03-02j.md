## Architecture Context

```
Current (before 02-02):
  MCP call_tool() → _get_storage() → KuzuBackend → handle_*(storage, args) → result

After 02-02:
  MCP call_tool() → _try_daemon_call(slug, tool, daemon_args) [best-effort, 5s timeout]
                  → success: return daemon result (no local KuzuBackend opened)
                  → fail (OSError/timeout): fallback to _dispatch_tool() with _get_storage()
```

**Key implementation details in 02-02-PLAN.md:**
- `_get_local_slug()` — reads `.axon/meta.json` slug from cwd
- `slug` = `arguments.get("repo") or _get_local_slug()` (repo= param takes precedence)
- `daemon_args` = args stripped of `repo` and `max_tokens` keys (daemon handles slug directly, truncation is MCP-side)
- `max_tokens` truncation: `result[:max_tokens] + "\n[truncated at N chars]"` if `len(result) > max_tokens`
